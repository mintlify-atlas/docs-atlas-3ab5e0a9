This guide walks you through launching Terarium on your local machine and completing your first end-to-end workflow: create a project, add a model, and run a simulation.

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) and [Docker Compose](https://docs.docker.com/compose/install/) installed
- An [OpenAI API key](https://platform.openai.com/api-keys) (required for AI-assisted features)
- Git

## Set up and launch

<Steps>
  <Step title="Clone the repository">
    Clone the Terarium repository and navigate to the `deploy` directory:

    ```bash
    git clone https://github.com/DARPA-ASKEM/terarium.git
    cd terarium/deploy
    ```
  </Step>

  <Step title="Create your environment file">
    Copy the environment template to create your local `.env` file:

    ```bash
    cp .env_template .env
    ```

    Open `.env` and add your OpenAI API key:

    ```bash
    # macOS
    local_host_name=host.docker.internal
    # Linux (uncomment the line below and comment out the macOS line above)
    # local_host_name=localhost

    # OpenAI connection details
    secret_openai_key=sk-...your-key-here...
    ```

    <Note>
      On Linux, set `local_host_name=localhost` instead of `host.docker.internal`. The template includes both options as comments.
    </Note>
  </Step>

  <Step title="Add hostname entries">
    Terarium routes internal traffic through named hosts for Keycloak (authentication) and MinIO (object storage). Add these entries to your hosts file so your browser can resolve them:

    **Linux / macOS** — edit `/etc/hosts`:

    ```bash
    sudo sh -c 'echo "127.0.0.1  keycloak" >> /etc/hosts'
    sudo sh -c 'echo "127.0.0.1  minio" >> /etc/hosts'
    ```

    **Windows** — add the same two lines to `C:\Windows\System32\drivers\etc\hosts` using a text editor run as Administrator:

    ```
    127.0.0.1  keycloak
    127.0.0.1  minio
    ```
  </Step>

  <Step title="Start all services">
    From the `deploy` directory, start the full stack in the background:

    ```bash
    docker compose up -d
    ```

    Docker Compose pulls all required images and starts them in dependency order. On first run, pulling images and initializing MinIO buckets may take several minutes.

    <Tip>
      Monitor startup progress with `docker compose logs -f hmi-client`. The UI container waits for `hmi-server` to pass its health check before it starts accepting traffic.
    </Tip>
  </Step>

  <Step title="Open the application">
    Once the containers are running, open your browser and navigate to:

    **[http://localhost:8080](http://localhost:8080)**

    Log in using one of the default accounts:

    | Username | Password  | Role  |
    |----------|-----------|-------|
    | adam     | admin123  | Admin |
    | ursula   | user123   | User  |

    <Warning>
      These credentials are for local development only. Do not expose this deployment to public networks without changing all default passwords.
    </Warning>
  </Step>
</Steps>

## First workflow: create a project and run a simulation

Once you are logged in, follow these steps to complete your first end-to-end workflow.

<Steps>
  <Step title="Create a project">
    On the home screen, click **New project** and give your project a name. Projects are the top-level container for all your models, datasets, documents, and workflows.
  </Step>

  <Step title="Add a model">
    Inside your project, open the **Resources** panel and click **Add model**. You can:

    - Upload a model file (AMR JSON, SBML, or STELLA format)
    - Extract a model from a scientific paper by uploading a PDF
    - Browse the built-in model catalog

    For a quick start, use the model catalog to add a pre-built SIR epidemiological model.
  </Step>

  <Step title="Open the workflow canvas">
    Click **New workflow** to open the visual canvas. Drag your model from the resources panel onto the canvas. This creates a model node.
  </Step>

  <Step title="Configure and simulate">
    1. Connect a **Model configuration** node to your model node. Adjust initial conditions and parameters as needed.
    2. Connect a **Simulate** node to the configuration node.
    3. Click **Run** on the Simulate node to execute the simulation.
    4. Results appear as a time-series chart directly on the canvas.
  </Step>
</Steps>
