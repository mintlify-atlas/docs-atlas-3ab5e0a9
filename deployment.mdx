### Task runners

Task runners are long-lived workers that consume jobs from RabbitMQ. Each runner handles a specific class of AI or scientific task.

| Container | Image | Task type |
|---|---|---|
| `gollm-taskrunner` | `ghcr.io/darpa-askem/gollm-taskrunner` | LLM-based knowledge extraction and model enrichment |
| `mira-taskrunner` | `ghcr.io/darpa-askem/mira-taskrunner` | MIRA model operations |
| `funman-taskrunner` | `ghcr.io/darpa-askem/funman-taskrunner` | FUNMAN constraint validation |
| `ocr-extraction-taskrunner` | `ghcr.io/darpa-askem/ocr-extraction-taskrunner` | PDF OCR and document extraction |

## Startup order and health checks

Docker Compose enforces startup ordering through health check dependencies. The dependency chain is:

```
postgres (healthy)
  â””â”€â”€ keycloak-init (completed)
        â””â”€â”€ keycloak (healthy)
  â””â”€â”€ migrate-spicedb (completed)
        â””â”€â”€ spicedb (healthy)
  â””â”€â”€ minio (healthy)
        â””â”€â”€ migrate-minio (completed)
  â””â”€â”€ elasticsearch (healthy)
  â””â”€â”€ rabbitmq (healthy)
  â””â”€â”€ redis (healthy)
        â””â”€â”€ hmi-server (healthy)
              â””â”€â”€ hmi-client (healthy)
```

`hmi-server` performs a health check against `http://hmi-server:3000/actuator/health` with up to 30 retries at 3-second intervals and a 5-minute start period. `hmi-client` and `terarium-docs` check `http://localhost:80/healthcheck`.

<Note>
  On first run, `migrate-minio` downloads and imports a large epidemiology knowledge graph (~several hundred MB) into the `askem-static-storage` bucket. Subsequent starts skip this step if the data already exists.
</Note>

## Environment variables

Configuration lives in `deploy/.env`. Copy `deploy/.env_template` to `deploy/.env` before starting.

### Required

| Variable | Description |
|---|---|
| `secret_openai_key` | OpenAI API key. Used by `gollm-taskrunner`, `ocr-extraction-taskrunner`, and `beaker`. |

### Platform configuration

| Variable | Default | Description |
|---|---|---|
| `local_host_name` | `host.docker.internal` | Hostname used by containers to reach the Docker host. Set to `localhost` on Linux. |

```bash
# .env (macOS)
local_host_name=host.docker.internal
secret_openai_key=sk-...

# .env (Linux)
local_host_name=localhost
secret_openai_key=sk-...
```

<Warning>
  All default passwords in the compose files (`admin123`, `postgres`, `terarium123`, etc.) are hardcoded for local development. Do not use this configuration in a production or internet-facing environment without replacing every default credential.
</Warning>

## Data volumes

Docker named volumes persist data between container restarts:

| Volume | Used by | Contents |
|---|---|---|
| `terarium-db` | `postgres` | Application and Keycloak database |
| `minio-data` | `minio` | Uploaded files, datasets, simulation results, documents |
| `elasticsearch-data` | `elasticsearch` | Full-text search index |

To reset all data and start fresh:

```bash
docker compose down -v
docker compose up -d
```

<Warning>
  The `-v` flag removes all named volumes. This permanently deletes all stored data including user accounts, projects, models, and simulation results.
</Warning>

## Useful commands

```bash
# Start all services in the background
docker compose up -d

# Stop all services (preserve data volumes)
docker compose down

# Tail logs for a specific service
docker compose logs -f hmi-server

# Check health status of all containers
docker compose ps

# Restart a single service
docker compose restart hmi-client

# Pull the latest images and restart
docker compose pull && docker compose up -d
```

<Tip>
  The RabbitMQ management UI is available at [http://localhost:15672](http://localhost:15672) (username: `terarium`, password: `terarium123`) if you expose port `15672` in `docker-compose-common.yml`. By default only port `5672` (AMQP) is mapped.
</Tip>
